# AI 기반 응급 의료 접근성 대시보드

## 1. 프로젝트 개요

### 프로젝트 명

AI 기반 지역별 응급 의료 접근성 대시보드 및 정책 시뮬레이터

### 목표

'응급기관별_시도별_접근성.ipynb'의 정적 분석 결과를 기반으로, 정책 결정자가 실시간으로 데이터를 탐색하고, 정책 변경(예: 응급 기관 신설)이 지역의 의료 접근성 및 취약성 클러스터에 미치는 영향을 AI를 통해 시뮬레이션 및 질의응답할 수 있는 대화형 웹 애플리케이션을 구축한다.

### 핵심 차별점

- **정적(Static) → 동적(Dynamic):** 기존의 정적 리포트(HTML)가 아닌, 사용자가 직접 데이터를 조작하고 결과를 예측하는 '시뮬레이터' 기능 제공.
- **범용(General) AI → 전문(Domain-Specific) AI:** 단순 정보 제공 챗봇이 아닌, '제공된 데이터(Baseline Data)'와 '시뮬레이션 결과'를 실시간으로 인지하고 분석하여 답변하는 '데이터 기반(Data-Aware)' AI 챗봇 탑재.

## 2. 핵심 기능

### 가. 응급 의료 접근성 대시보드 (Baseline Dashboard)

- **목적:** 현재 17개 시도의 응급 의료 접근성 현황을 시각적으로 파악.
- **데이터:** `.ipynb`의 K-Means 클러스터링 분석 결과(클러스터 A, B, C) 및 주요 지표(고령화율, 인구 10만 명당 기관 수)를 JSON 형태로 웹에 내장.
- **시각화 (Chart.js 활용):**
    - **시도별 접근성 비교:** 지역별 '인구 10만 명당 응급 기관 수' 막대그래프.
    - **취약 지역 분포:** 클러스터(A, B, C)별 지역 분포 파이 차트.
    - **클러스터 특징:** 클러스터별 평균 지표(고령화율, 기관별 접근성)를 보여주는 히트맵 또는 테이블. (ipynb의 히트맵과 동일)

### 나. AI 정책 시뮬레이터 (Policy Simulator)

- **목적:** 특정 지역에 응급 의료 자원(기관)을 추가/감소시켰을 때, 해당 지역의 접근성 지표와 취약성 클러스터(A, B, C)가 어떻게 변화하는지 예측.
- **작동 로직:**
    1. **입력:** 사용자가 '시도' (예: 강원)와 '정책' (예: 지역응급의료센터 +5개)을 선택.
    2. **재계산:** 앱이 '강원'의 인구 데이터를 기반으로 변경된 '인구 10만 명당 지역응급의료센터' 수치를 실시간으로 재계산.
    3. **클러스터 재분류 (AI):**
        - `.ipynb`에서 도출된 기존 3개 클러스터의 중심값(Centroids)을 미리 저장.
        - 변경된 '강원'의 새 지표와 각 클러스터 중심값 간의 '유클리드 거리'를 계산.
        - '강원'을 거리가 가장 가까운 클러스터(A, B, C)로 재분류.
    4. **결과 출력:** "정책 실행 전 (Cluster A) → 정책 실행 후 (Cluster B)"와 같이 변화된 내용을 즉각적으로 표시.

### 다. 데이터 기반 Q&A 챗봇 (Data-Aware Chatbot)

- **목적:** 사용자가 대시보드 및 시뮬레이션 결과에 대해 자연어로 질문하고, 데이터에 근거한 답변을 얻음.
- **AI 모델:** Gemini API (Google Search Grounding 포함)
- **작동 로직:**
    1. **데이터 주입:** 사용자가 질문할 때마다, (1) 현재 대시보드의 Baseline 데이터 + (2) 방금 실행한 시뮬레이션 결과 + (3) 클러스터 특징 데이터를 AI 프롬프트의 '컨텍스트(Context)'로 함께 주입.
    2. **시스템 프롬프트 (지시):** "당신은 한국 응급 의료 정책 분석 전문가입니다. 반드시 내가 제공한 '컨텍T스트 데이터'와 '시뮬레이션 결과'에 기반하여 답변해야 합니다. 외부 정보를 묻는다면 Google 검색을 활용할 수 있습니다."
    3. **질의응답 예시:**
        - **[단순 데이터 조회]** "Cluster A(농어촌형)에 속한 지역은 어디인가?" → (AI가 데이터를 보고) "전남, 경북, 강원 등입니다."
        - **[데이터 해석]** "서울이 Cluster C(대도시형)인 이유는?" → (AI가 데이터를 보고) "고령화율(15.6%)은 낮지만, 인구 대비 응급시설 수(0.25개/10만명)가 매우 낮기 때문입니다."
        - **[시뮬레이션 해석]** "방금 내가 강원에 센터 5개를 추가한 시뮬레이션 결과는 어땠나?" → (AI가 시뮬레이션 결과를 보고) "강원의 '지역센터 접근성'이 0.74에서 1.07로 증가하여, 취약성 등급이 'Cluster A'에서 'Cluster B'로 개선되었습니다."

## 3. 기대 효과

- **논문 기여:** 정적 분석(ipynb)을 동적 의사결정 보조 도구(Web App)로 발전시켜 연구의 활용성을 입증.
- **실용성:** 정책 결정자가 다양한 시나리오를 테스트하고, AI의 데이터 기반 조언을 얻어 근거 중심의 정책을 수립하도록 보조.
